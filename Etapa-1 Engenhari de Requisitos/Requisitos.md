# Elicitação e Estruturação de Requisitos

## Requisitos Funcionais (RF)

### Must Have
- RF1: O sistema deve permitir o cadastro de quartos com todos os campos especificados.
- RF2: O sistema deve listar todos os quartos com suas informações e disponibilidade.
- RF3: O sistema deve permitir editar informações de quartos.
- RF4: O sistema deve permitir o cadastro de hóspedes com nome, sobrenome, CPF e e-mail.
- RF5: O sistema deve listar todos os hóspedes cadastrados.
- RF6: O sistema deve permitir a criação e edição de reservas, associando hóspedes a quartos.
- RF7: O sistema deve exibir a disponibilidade dos quartos (Ocupado, Livre, Manutenção e Limpeza).

### Should Have
- RF8: O sistema deve permitir filtrar quartos por disponibilidade e tipo.
- RF9: O sistema deve permitir filtrar hóspedes por nome ou CPF.

### Could Have
- RF10: O sistema pode permitir exportação de listas de quartos e hóspedes.
- RF11: O sistema pode permitir visualização de histórico de reservas.

### Won’t Have
- RF12: Integração com sistemas externos de pagamento (não será implementado nesta versão).

---

## Requisitos Não Funcionais (RNF)

### Must Have
- RNF1: O sistema deve possuir interface web responsiva.
- RNF2: O sistema deve utilizar paleta de cores verde e azul.
- RNF3: O sistema deve garantir segurança dos dados dos hóspedes (ex: proteção de CPF e e-mail).
- RNF4: O sistema deve ser compatível com os principais navegadores (Chrome, Firefox, Edge).

### Should Have
- RNF5: O sistema deve ter desempenho adequado para até 100 usuários simultâneos.
- RNF6: O sistema deve possuir documentação de uso.

### Could Have
- RNF7: O sistema pode permitir personalização de temas pelo usuário.

### Won’t Have
- RNF8: Suporte a dispositivos móveis nativos (aplicativo mobile) não será implementado nesta versão.
